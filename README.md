Распределенное выполнение и сетевое взаимодействие.

Финальная модификация программы предполагает переход к параллельному функционированию агентов на различных вычислительных узлах с организацией сетевого взаимодействия (в том числе через брокер запросов).

Реализованные классы: 

Broker: Центральный координатор, который запускает TCP-сервер, управляет распределенным рекурсивным поиском решений через агентов, проверяет совместимость позиций с использованием KnowledgeBase и сохраняет найденные решения.

AgentInfo: Структура для хранения всей информации о подключенном агенте, включая ID, сокет, текущую строку, флаг фиксации, приоритет и статус активности.

BrokerMessage: Структура, определяющая формат и типы сообщений (регистрация, запрос позиции, принятие/отказ, откат, найденное решение, heartbeat и др.) для обмена между брокером и агентами.

AgentClient: Клиентская часть агента, управляющая TCP-соединением с брокером, отправляющая запросы позиций и обрабатывающая ответы в конечном автомате состояний.

BrokerWindow: Главное окно брокера для запуска/остановки сервера, автоматического запуска локальных агентов через QProcess, отображения таблицы агентов, визуализации доски и навигации по найденным решениям.

AgentWindow: Окно агента для подключения к брокеру, отображения текущей позиции на доске, управления поиском, установки приоритета и запроса фиксации позиции.

QueenAgent: Класс локального агента-ферзя, который знает только своего соседа слева, проверяет атаки и двигается по доске с поддержкой базы знаний (веса и цвета) и фиксированных позиций.

QueensSolver: Класс для локального рекурсивного решения задачи о N ферзях с битовыми масками, проверкой цветовой совместимости из KnowledgeBase и сортировкой решений по весу.

ChessBoardWidget: Виджет отображения шахматной доски с полупрозрачными цветами базы знаний, весами клеток, ферзями-кругами (с разным цветом для фиксированных) и числами приоритетов.

PriorityManager: Менеджер, хранящий приоритеты и фиксированные позиции для столбцов, а также определяющий порядок перебора столбцов (фиксированные → по приоритету → по ID).

PrioritySpinBox: Специализированный виджет над столбцом для отображения приоритета (1-10), иконки замка при фиксации и звездочки для высокого приоритета с поддержкой двойного клика для редактирования.

KnowledgeBase: База знаний на SQLite для хранения веса (1-100) и цвета (Red/Green/Blue) каждой клетки доски с методами получения/установки, проверки цветовой совместимости и вычисления суммарного веса решения.

KnowledgeBaseDialog: Диалоговое окно с таблицей N×N для просмотра и редактирования весов и цветов всех клеток, панелью статистики и функциями генерации случайных распределений (все зеленые, 2 цвета 50/50, 3 цвета).

SortDialog: Диалоговое окно с выбором направления сортировки решений (по возрастанию или убыванию веса), отправляющее сигнал с параметром ascending.
